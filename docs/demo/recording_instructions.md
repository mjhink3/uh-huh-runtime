# Uh-Huh Runtime V0.1 Recording Instructions

## What Was Created

This environment does not have a usable screen recording stack available, so this folder contains a recording-ready fallback package:

- `terminal_transcript.txt`: exact terminal output from the demo commands.
- `recording_audit.jsonl`: audit ledger generated by the demo commands.
- `voiceover_script.txt`: ElevenLabs-ready narration.
- `demo_captions.srt`: subtitle file.
- `01_title.png` through `06_close.png`: static terminal-style frames for editing.
- `storyboard.html`: browser-viewable storyboard.
- `edit_decision_list.csv`: timing guide for Clipchamp, OBS, CapCut, or any editor.

## Recommended Manual Recording Path

1. Open PowerShell.
2. Run:

```powershell
cd C:\Users\mjhin\uh_huh_runtime_v01
powershell -ExecutionPolicy Bypass -File .\demo_recording.ps1
```

3. Start screen recording with OBS, Clipchamp, or Windows Snipping Tool before pressing Enter.
4. Press Enter between sections when the narration reaches the next scene.
5. Stop recording after the closing "Demo complete" screen.
6. Generate voiceover in ElevenLabs using `docs/demo/voiceover_script.txt`.
7. Import the screen recording and voiceover into Clipchamp/CapCut.
8. Optional: import `docs/demo/demo_captions.srt` as captions.
9. Export as:

```text
docs/demo/uh-huh-runtime-v0.1-demo.mp4
```

## Fallback Edit Without Screen Recording

If screen recording is unavailable, create a simple video from the PNG frames:

1. Import `01_title.png` through `06_close.png` into Clipchamp or CapCut.
2. Use the durations in `edit_decision_list.csv`.
3. Add the ElevenLabs voiceover from `voiceover_script.txt`.
4. Add captions from `demo_captions.srt`.
5. Export as MP4.
